================================================================================
          LAPORAN DESAIN ANTARMUKA SISTEM: MASUKAN DAN KELUARAN SQUASHY
================================================================================

Laporan ini menyajikan analisis desain antarmuka sistem E-Commerce Squashy 
yang dibagi menjadi dua bagian utama: 4.3 Desain Masukan (Input Design) dan 
4.4 Desain Keluaran (Output Design). Setiap bagian dikelompokkan berdasarkan 
peran pengguna, yaitu sisi Pembeli (Customer) dan sisi Penjual (Admin/Owner). 
Analisis ini disusun berdasarkan berkas kode sumber yang ada pada sistem Squashy 
(dengan mengecualikan halaman utama/beranda sesuai ketentuan).

Setiap penjelasan halaman diawali dengan keterangan penempatan gambar (placeholder 
gambar) untuk kebutuhan dokumentasi visual sistem.

================================================================================
4.3 DESAIN MASUKAN (INPUT DESIGN)
================================================================================

Desain masukan dirancang untuk menangkap data, pilihan, dan instruksi dari 
pengguna secara efektif, aman, dan interaktif menggunakan elemen formulir (form), 
kolom input, dropdown, area teks, checkbox, dan pengunggah berkas.

--------------------------------------------------------------------------------
4.3.1 Sisi Pembeli (Customer Side)
--------------------------------------------------------------------------------

a. Registrasi Akun Pembeli Baru (customers/register.php)

gambar 1 Form Registrasi Akun Pembeli Baru

gambar 1 Form Registrasi Akun Pembeli Baru adalah formulir input bagi pengunjung 
baru untuk mendaftarkan akun pembeli agar dapat mengakses fitur keranjang, 
checkout, profil, ulasan, dan chat pada sistem Squashy.
Konten Utama (Area Kerja) Area utama berpusat pada kartu registrasi putih 
berbingkai hijau tebal dengan logo Squashy di atasnya.
1. Formulir Registrasi (Input Field):
   o Kolom Nama Lengkap: Text field bersimbol ikon 👤 untuk menginput nama lengkap asli pembeli.
   o Kolom Alamat Email: Email input field bersimbol ikon ✉️ untuk mendaftarkan alamat email unik (contoh placeholder: Dummy123@gmail.com).
   o Kolom Buat Kata Sandi: Password field bersimbol ikon 🔒 untuk memasukkan sandi rahasia yang karakternya akan disamarkan.
   o Kolom Konfirmasi Kata Sandi: Password field bersimbol ikon 🔒 untuk memasukkan ulang sandi guna memastikan kecocokan sandi.
   o Kolom Tanggal Lahir: Date field bersimbol ikon 📅 yang akan membuka kalender pop-up saat diklik untuk merekam tanggal lahir pengguna.
   o Checkbox Syarat & Ketentuan: Checkbox interaktif di bagian bawah input yang wajib dicentang sebagai tanda persetujuan ketentuan layanan.
   o Tujuan: Menangkap data identitas pembeli baru secara lengkap dan valid untuk disimpan dalam tabel user.

2. Tombol Aksi Submit:
   o Tombol "DAFTAR SEKARANG": Tombol pink lebar dengan efek timbul 3D shadow saat ditekan.
   o Fungsi: Mengirimkan seluruh data masukan registrasi ke database, memicu enkripsi hash SHA-256 pada sandi, dan mengalihkan pembeli ke halaman login setelah registrasi berhasil.
   o Tujuan: Mengeksekusi proses pembuatan akun baru pada database.


b. Login Autentikasi Pengguna (customers/login.php)

gambar 2 Form Login Pengguna

gambar 2 Form Login Pengguna adalah formulir autentikasi masuk sistem yang digunakan 
baik oleh pembeli terdaftar maupun admin Squashy untuk membuka sesi aktif masing-masing.
Konten Utama (Area Kerja) Area utama berfokus pada kartu login putih berbingkai 
kuning dengan maskot logo Squashy di atasnya.
1. Formulir Login (Input Field):
   o Kolom Username (Email/Nama Admin): Text field bersimbol ikon 👤 untuk menangkap alamat email pembeli atau username nama admin.
   o Kolom Kata Sandi: Password field bersimbol ikon 🔒 untuk memasukkan kata sandi rahasia dengan karakter tersamarkan (masked text).
   o Tujuan: Menangkap kredensial login pengguna secara aman.

2. Tombol Aksi & Navigasi:
   o Tombol "MASUK SEKARANG": Tombol oranye lebar melengkung dengan efek 3D shadow.
   o Tautan "Lupa Kata Sandi?": Link teks abu-abu di bawah tombol masuk untuk mengalihkan pengguna ke halaman reset password.
   o Tautan "Daftar di sini": Link teks biru muda untuk pengunjung yang belum memiliki akun agar beralih ke register.php.
   o Fungsi: Mengirimkan kredensial masuk untuk diverifikasi kecocokannya dengan data terenkripsi di tabel user atau admin, kemudian mengaktifkan sesi login (session).
   o Tujuan: Memverifikasi hak akses pengguna untuk membuka fitur-fitur belanja atau panel dashboard admin.


c. Reset Password (customers/forgot_password.php)

gambar 3 Form Reset Password

gambar 3 Form Reset Password digunakan ketika pengguna lupa kata sandi akunnya 
dan ingin memperbaruinya secara mandiri dengan memasukkan email terdaftar.
Konten Utama (Area Kerja) Tampilan kartu form putih berbingkai kuning dengan border 
melengkung 30px di tengah halaman berlatar belakang awan pastel.
1. Formulir Reset (Input Field):
   o Kolom Alamat Email: Email input field bersimbol ikon ✉️ tempat pengguna menginput email pemulihan terdaftar.
   o Kolom Kata Sandi Baru: Password field bersimbol ikon 🔒 tempat pengguna menginput sandi pengganti yang baru.
   o Tujuan: Mengidentifikasi email target dan menangkap nilai sandi baru yang akan disimpan.

2. Tombol Aksi:
   o Tombol "RESET PASSWORD": Tombol jingga lebar dengan efek bayangan penekanan.
   o Fungsi: Mencari kecocokan email di database, mengubah kata sandi lama dengan password baru terenkripsi hash SHA-256, memunculkan notifikasi sukses, dan mengarahkan kembali ke halaman login.
   o Tujuan: Memperbarui record kata sandi pengguna secara aman di tabel user.


d. Pengaturan Kuantitas & Hapus Item Keranjang (customers/keranjang.php)

gambar 4 Input Kuantitas dan Aksi Hapus Keranjang Belanja

gambar 4 Input Kuantitas dan Aksi Hapus Keranjang Belanja adalah kontrol masukan 
yang disematkan pada baris daftar produk belanja di halaman keranjang belanja 
pembeli untuk memodifikasi daftar belanjaan sebelum melakukan checkout.
Konten Utama (Area Kerja) Area baris produk di dalam kartu keranjang belanja putih.
1. Kontrol Kuantitas Produk:
   o Tombol Minus (-): Tombol bulat pink kecil berskala mikro untuk mengurangi kuantitas belanja produk.
   o Tombol Plus (+): Tombol bulat pink kecil berskala mikro untuk menambah kuantitas belanja produk.
   o Nilai Kuantitas: Teks nilai angka kuantitas yang diapit oleh tombol minus dan plus, dibatasi minimal 1 dan maksimal jumlah stok produk yang tersedia di gudang.
   o Fungsi: Memicu update data kuantitas barang belanjaan secara langsung ke tabel keranjang (cart) di database.
   o Tujuan: Memberikan kontrol dinamis bagi pembeli untuk menyesuaikan jumlah barang anak yang ingin dibeli.

2. Aksi Penghapusan Item:
   o Tombol "🗑️ Hapus": Tombol merah di sisi kanan baris produk.
   o Fungsi: Mengaktifkan dialog konfirmasi bawaan browser untuk menghapus record item belanja tersebut dari database keranjang belanja user.
   o Tujuan: Membuang item yang batal dibeli dari daftar belanjaan pembeli.


e. Form Checkout & Detail Alamat Pengiriman (customers/checkout.php)

gambar 5 Form Detail Pengiriman dan Opsi Pembayaran Checkout

gambar 5 Form Detail Pengiriman dan Opsi Pembayaran Checkout adalah formulir masukan 
bagi pembeli untuk melengkapi informasi pengiriman fisik barang serta memilih kurir ekspedisi.
Konten Utama (Area Kerja) Kartu putih lebar berbayang di kolom kiri halaman checkout.
1. Input Informasi Penerima:
   o Kolom Nama Pembeli: Text field tempat menginput nama lengkap penerima paket belanjaan.
   o Kolom Nomor HP: Text field berfitur inputmode="numeric" dan pembatasan pattern angka dengan panjang maksimal 20 digit, untuk menangkap nomor telepon aktif penerima.
   o Tujuan: Menangkap data kontak penerima barang secara valid.

2. Pilihan Dropdown Wilayah & Kurir (Integrasi RajaOngkir):
   o Dropdown Provinsi Tujuan: Pilihan provinsi tujuan pengiriman yang datanya diambil dinamis dari API RajaOngkir.
   o Dropdown Kota / Kabupaten Tujuan: Pilihan kota/kabupaten tujuan yang terhubung dinamis dan otomatis terbuka setelah provinsi dipilih.
   o Dropdown Pilih Kurir Ekspedisi: Pilihan kurir yang tersedia (JNE, TIKI, POS Indonesia) yang otomatis aktif setelah kota tujuan dipilih.
   o Dropdown Pilih Layanan Pengiriman: Pilihan paket ongkir beserta estimasi hari (contoh: JNE OKE, REG, YES) yang muncul setelah kurir dipilih.
   o Tujuan: Menentukan destinasi pengiriman logistik secara akurat guna menghitung tarif ongkos kirim asli.

3. Input Alamat Fisik Lengkap:
   o Kolom Alamat Lengkap: Textarea luas untuk menuliskan detail jalan, RT/RW, nomor rumah, dan patokan pengiriman secara spesifik.
   o Tujuan: Menangkap alamat detail tujuan pengantaran kurir fisik.

4. Tombol Submit "Proses Pembayaran":
   o Fungsi: Memvalidasi keharusan pengisian seluruh form, menyimpan data transaksi baru ke tabel order dengan status "pending", memicu pop-up pembayaran online Midtrans Snap, serta mengosongkan keranjang belanja pembeli.
   o Tujuan: Menyelesaikan pengisian data transaksi dan memicu pelunasan pembayaran.

gambar 6 Form Tulis Ulasan dan Rating Produk

gambar 6 Form Tulis Ulasan dan Rating Produk adalah formulir feedback pembeli

   o Pemilih Rating Bintang: 5 radio button berbentuk bintang emas CSS interaktif yang menyala saat disorot (hover) dari kanan ke kiri, mempresentasikan rating 1 hingga 5 bintang.

   o Unggah Bukti Foto: Input file dengan pembatasan menerima berkas gambar (accept="image/*") untuk mengunggah foto bukti fisik produk asli.
   o Tujuan: Menangkap ulasan tertulis, rating bintang, dan berkas foto ulasan dari pembeli.

2. Tombol Kirim Ulasan:
   o Tombol "Kirim Ulasan 💖": Tombol submit pink lebar di bagian bawah formulir.
   o Fungsi: Mengunggah berkas foto ulasan ke server, merekam data ke tabel review_produk di database, memicu ulasan agar tampil publik di modal produk, dan mengarahkan kembali pembeli ke profilnya.
   o Tujuan: Menyimpan ulasan ke database dan mempublikasikannya.

a. Modal Tambah & Edit Katalog Produk (admin/kelola_produk.php)

gambar 8 Modal Input Tambah dan Edit Produk Admin

gambar 8 Modal Input Tambah dan Edit Produk Admin adalah form pop-up modal 
terintegrasi untuk memfasilitasi admin dalam memanipulasi persediaan barang di toko.
Konten Utama (Area Kerja) Jendela pop-up berlatar belakang krem yang menutupi area 
kerja tabel produk saat tombol "Tambah Produk Baru" atau tombol "Edit" diklik.
1. Formulir Data Produk (Input Field):
   o Kolom Nama Produk: Text field untuk menginput nama barang mainan, pakaian, atau perlengkapan anak.
   o Kolom SKU/Kode: Text field untuk memasukkan kode stock keeping unit barang.
   o Dropdown Kategori: Select option berisi daftar kategori yang ada (Mainan, Pakaian, Perlengkapan).
   o Kolom Harga: Number field untuk menentukan nominal harga jual barang (minimal 0).
   o Kolom Stok: Number field untuk menentukan kuantitas unit stok barang di gudang (minimal 0).
   o Area Deskripsi Singkat: Textarea opsional untuk menuliskan spesifikasi detail barang.
   o Tujuan: Menangkap data spesifikasi dan stok produk yang akan ditayangkan.

2. Area Pengunggah Gambar:
   o Tombol "Pilih Gambar" dan input file tersembunyi (images[]): Mengunggah berkas foto produk (mendukung multi-upload hingga 4 gambar) dengan box pratinjau (thumbnail preview) gambar instan di bawahnya.
   o Tujuan: Mengunggah file visual produk ke server.

3. Tombol Aksi Modal:
   o Tombol "BATALKAN": Tombol abu-abu untuk menutup modal dan meriset seluruh isian form.
   o Tombol "SIMPAN/UPDATE PRODUK": Tombol oranye untuk mengirim data form melalui AJAX ke script produk_action.php.
   o Fungsi: Menyimpan produk baru atau memperbarui data produk lama di tabel produk dan memicu pembaruan badge status ketersediaan secara otomatis.
   o Tujuan: Memproses manipulasi data produk di database.

4.4 DESAIN KELUARAN (OUTPUT DESIGN)
================================================================================

Desain keluaran dirancang untuk memformat, menyusun, dan menampilkan informasi 
sistem Squashy kepada pengguna secara estetik, responsif, dan mudah dibaca, 
baik dalam bentuk katalog produk, jendela riwayat pesanan, invoice, visual 
grafik pendapatan, maupun tabel database operasional admin.

4.4.1 Sisi Pembeli (Customer Side)
--------------------------------------------------------------------------------

a. Katalog Etalase Kategori Produk (customers/kategori.php)

gambar 13 Halaman Katalog Kategori Produk

gambar 13 Halaman Katalog Kategori Produk adalah halaman output utama pembeli 
untuk menelusuri katalog produk Squashy yang disaring berdasarkan kategori aktif.
Konten Utama (Area Kerja) Area grid layout kartu produk dengan dynamic theme 
warna latar belakang yang berubah selaras dengan kategori aktif (Mainan: hijau; 
Pakaian: kuning; Perlengkapan: merah; Semua: biru).
1. Spanduk Judul Nunito:
   o Teks "KATEGORI SQUASHY": Judul halaman berukuran 60px putih dengan drop shadow tebal yang memberikan kesan ceria ramah anak.

2. Filter Tab Kategori (Pill Tabs):
